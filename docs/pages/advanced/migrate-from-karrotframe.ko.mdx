# Karrotframe에서 이사가기

Karrotframe과 **Stackflow**는 다음과 같은 멘탈모델의 차이가 존재해요.

Karrotframe의 경우,

1. Path를 기반으로 라우팅해요. 필요한 파라미터가 있다면 쿼리 파라미터에 담아요.
2. 기본 UI를 포함하고 있어요. 따라서 각 페이지 컴포넌트는 껍데기에 담길 내용만 포함해요.

**Stackflow**의 경우,

1. 액티비티 이름과 액티비티 파라미터 객체를 기반으로 라우팅해요.
2. 기본 UI를 포함하고 있지 않아요. 따라서 각 액티비티 컴포넌트는 basic-ui 등 껍데기도 포함해야해요.

이러한 멘탈 모델을 옮겨간다고 생각하고 이사를 하면 좋아요.

<Callout type="warning" emoji="⚠️">
  아래 가이드는 부분적인 컨셉을 담고있고 100% 기능을 커버하지 않아요.
  Karrotframe에서 쓰던 기능은 전부 다 옮겨왔으므로, 혹시 추가적으로 필요하다고
  생각되는 기능이 있으시다면 문서를 찬찬히 살펴보시는걸 추천해요.
</Callout>

## 1. Navigator를 Stack으로 옮겨가기

다음과 같이 기존의 `<Navigator />`와 `<Screen />` 컴포넌트를 통해 페이지를 등록하던 로직을 `stackflow()` 함수와 `historySyncPlugin()`을 통해 등록하는 과정으로 바꿔요.

```tsx
/**
 * as-is:
 */
const App: React.FC = () => {
  return (
    <Navigator>
      <Screen path="/my-page/:someid" component={MyPage} />
      <Screen path="*" component={NotFoundPage} />
    </Navigator>
  );
};
```

```tsx
/**
 * to-be:
 */
const { Stack } = stackflow({
  activities: {
    MyPage,
    NotFoundActivity,
  },
  plugins: [
    historySyncPlugin({
      routes: {
        MyPage: "/my-page/:someid",
      },
      fallbackActivity: () => "NotFoundActivity",
    }),
  ],
});

const App: React.FC = () => {
  return <Stack />;
};
```

## 2. `useNavigator()`와 `useParams()`를 `useFlow()`, `useActivityParams()`로 교체하기

다음과 같이 기존에 네비게이팅을 담당하던 `useNavigator()`를 `useFlow()`로 변경해요.

```tsx
/**
 * as-is:
 */
const MyPage = () => {
  const { push } = useNavigator();

  const onClick = () => {
    push("/something-page?foo=bar");
  };

  return <div>{/* ... */}</div>;
};
```

```tsx
/**
 * to-be:
 */
const MyPage = () => {
  const { push } = useFlow();

  const onClick = () => {
    push("SomethingPage", {
      foo: "bar",
    });
  };

  return <div>{/* ... */}</div>;
};
```

기존에 `useParams()` 훅으로 파라미터를 받던 부분을 `useActivityParams()`로 교체해요.

```tsx
/**
 * as-is:
 */
const SomethingPage = () => {
  const { foo } = useParams();

  return <div>{/* ... */}</div>;
};
```

```tsx
/**
 * to-be:
 */
const SomethingPage = () => {
  const { foo } = useActivityParams();

  return <div>{/* ... */}</div>;
};
```

## 3. 각 루트 컴포넌트에서 `@stackflow/basic-ui`의 `<AppScreen />` 컴포넌트를 사용하기

Stackflow는 기본적으로 UI를 포함하고 있지 않아요. 따라서 껍데기 컴포넌트를 각 액티비티에 넣어주어야해요.

```tsx
/**
 * as-is:
 */
const SomethingPage = () => {
  return <div>Hello, World!</div>;
};
```

```tsx
/**
 * to-be:
 */
const SomethingPage = () => {
  const { foo } = useActivityParams();

  return (
    <AppScreen theme="cupertino">
      <div>Hello, World!</div>
    </AppScreen>
  );
};
```

<Callout emoji="⚡️">
  `<AppScreen />` 컴포넌트에 필요한 Props는 여러 부분에서 반복될 수 있어요.
  따라서 `<AppScreen />
  `을 `<MyLayout />` 같은 컴포넌트로 한번 감싸서 활용하시는걸 추천드려요.
</Callout>
